Python Basics
Variables, functions, loops, conditionals, file handling
PythonSyntaxFile I/OData Types
▾

Python is the dominant language for computer vision and machine learning because of its readable syntax, rich library ecosystem and interactive development style. Before using any CV or ML library, the fundamentals must be solid.

Core Language Concepts
01
Variables and Data Types: Python is dynamically typed. Understanding when something is an int, float, string, list, dict or numpy array prevents the majority of beginner bugs in data pipelines.
02
Functions: Encapsulate logic with def. Use arguments and return values to build reusable preprocessing steps, each doing one clear job.
03
Loops and Comprehensions: for loops iterate over image batches, video frames, and dataset files. List comprehensions are faster and more Pythonic than equivalent for-loops for transformations.
04
File Handling: Reading image paths with open(), os.listdir(), and glob is how datasets are loaded. Always use with open(...) to guarantee files are closed even if an error occurs.
Code — File Handling and os.path
Python
import os
from glob import glob

# List all .jpg files in a dataset directory
image_dir  = "dataset/images/"
image_paths = glob(os.path.join(image_dir, "*.jpg"))

print(f"Found {len(image_paths)} images")

# Read a text label file safely
with open("labels.txt", "r") as f:
    labels = [line.strip() for line in f.readlines()]

# Write results to a file
with open("results.txt", "w") as f:
    for path, label in zip(image_paths, labels):
        f.write(f"{path}: {label}\n")

NumPy and Matplotlib
Array operations, vectorised math, and visualising data and images
NumPyMatplotlibArraysVisualisation
▾

NumPy is the foundation of all scientific Python. Images are NumPy arrays. Every pixel value, every bounding box coordinate, every model prediction is a NumPy array operation at some level. Matplotlib lets you visualise these arrays — essential for debugging pipelines.

NumPy
N-dimensional array library. Operations are vectorised — running on C under the hood, thousands of times faster than Python loops. Core object is np.ndarray. Images are 3D arrays: (height, width, channels).
Matplotlib
Plotting library. plt.imshow() shows images. plt.hist() shows pixel distributions. plt.subplot() compares before/after. Essential for understanding what preprocessing does to your data.
image.shape = (H, W, C)  — Height × Width × Channels
pixel range: uint8 = [0, 255]    float32 = [0.0, 1.0]
NumPy dtype matters. OpenCV reads uint8. Neural networks expect float32 in [0, 1]. Always normalise before training.

Images as Matrices
RGB vs Grayscale, pixel values, channels, and how computers see images
OpenCVPixelsRGBGrayscaleMatrix
▾

A computer does not see a picture — it sees a matrix of numbers. Each number is a pixel value from 0 (black) to 255 (white) for 8-bit images. A colour image has three such matrices stacked together, one per channel. Understanding this is the gateway to all computer vision.

Grayscale Image
2D matrix: shape (H, W). Each value 0–255 represents brightness. 0 = black, 255 = white. One channel only. Used in edge detection and where colour is irrelevant. Memory is 1/3 of colour.
Colour Image (BGR/RGB)
3D matrix: shape (H, W, 3). Three channels — Blue, Green, Red in OpenCV (BGR). Red, Green, Blue in most other systems (RGB). Mixing up BGR/RGB is the most common beginner mistake.
RGB pixel: (R, G, B) = (255, 0, 0)  →  Pure Red
Grayscale: Y = 0.299R + 0.587G + 0.114B
Human eyes are most sensitive to green — that is why green has the highest weight in the grayscale formula

Code — Preprocessing Pipeline
Python — OpenCV
import cv2
import numpy as np

def preprocess_for_yolo(img_path, target_size=640):
    img = cv2.imread(img_path)

    # Resize to square (YOLOv8 expects square input)
    img = cv2.resize(img, (target_size, target_size))

    # Convert BGR to RGB
    img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)

    # Normalise to [0, 1]
    img = img.astype(np.float32) / 255.0
    return img

# Canny edge detection
def detect_edges(img_path, low=50, high=150):
    img   = cv2.imread(img_path)
    gray  = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    blur  = cv2.GaussianBlur(gray, (5, 5), 0)  # reduce noise first
    edges = cv2.Canny(blur, low, high)
    return edges

Detection vs Classification and Bounding Boxes
What each task means, bounding box format and confidence scores
DetectionClassificationBounding BoxConfidenceIoU
▾

These two tasks are often confused. Classification answers "what is in this image?" with a single label. Detection answers "what is in this image AND where is it?" — returning both a label and a bounding box for every object found.

Image Classification
Input: full image. Output: one class label (+ confidence). Example: "cat — 94%". Cannot locate the object or handle multiple objects. Used when the whole image depicts one thing.
Object Detection
Input: full image. Output: list of (class, confidence, bounding box) for every detected object. Can handle many objects of many classes simultaneously. Used for most real-world robotics applications.
Bounding box: (xcenter, ycenter, width, height)  — YOLO format
IoU = Area of Overlap ÷ Area of Union    (0 = no overlap, 1 = perfect)
IoU (Intersection over Union) measures how well a predicted box matches the ground-truth box. Threshold typically 0.5 for a detection to count as correct.

Code — YOLOv8 Object Detection
Python — YOLOv8
from ultralytics import YOLO
import cv2

model = YOLO("yolov8n.pt")  # nano model — fastest

results = model.predict(source="video.mp4", conf=0.5, save=True)

for result in results:
    boxes = result.boxes      # all detected bounding boxes
    for box in boxes:
        x1, y1, x2, y2 = box.xyxy[0]   # corner format
        conf  = box.conf[0]              # confidence score
        cls   = int(box.cls[0])          # class index
        label = model.names[cls]          # class name string
        print(f"{label}: {conf:.2f} at [{x1:.0f},{y1:.0f},{x2:.0f},{y2:.0f}]")

Object Tracking across Video Frames
Maintaining object identity across frames with DeepSORT and OpenCV trackers
TrackingDeepSORTOpenCVObject IDKalman Filter
▾

Detection runs on a single frame. Tracking links detections across frames so that the same object keeps the same ID as it moves through a video. Without tracking, a person detected in frame 1 and frame 2 would be two different detections with no connection between them.

Detection vs Tracking Pipeline
Frame N
video frame
▶
YOLOv8
detect objects
▶
DeepSORT
assign / update IDs
▶
Output
ID + box per object
Kalman Filter
Predicts where each tracked object will be in the next frame based on its velocity. This prediction is matched against new detections. Handles short occlusions where the object briefly disappears.
Hungarian Algorithm
Solves the assignment problem: which detection in frame N+1 corresponds to which track from frame N? Minimises total cost (usually based on IoU between predicted and detected boxes).
Re-ID Features
DeepSORT adds a deep appearance descriptor (like a face/body embedding) per detection. This allows re-identification after long occlusion when Kalman prediction alone would fail.
Code — YOLOv8 + Built-in Tracker
Python — YOLOv8 Tracking
from ultralytics import YOLO
import cv2

model = YOLO("yolov8n.pt")
cap   = cv2.VideoCapture("video.mp4")

while cap.isOpened():
    ret, frame = cap.read()
    if not ret:
        break

    # .track() runs detection + tracking on every frame
    results = model.track(frame, persist=True, tracker="bytetrack.yaml")

    if results[0].boxes.id is not None:
        ids    = results[0].boxes.id.int().tolist()
        boxes  = results[0].boxes.xyxy
        for obj_id, box in zip(ids, boxes):
            x1,y1,x2,y2 = map(int, box)
            cv2.rectangle(frame, (x1,y1), (x2,y2), (0,255,0), 2)
            cv2.putText(frame, f"ID {obj_id}", (x1,y1-10),
                       cv2.FONT_HERSHEY_SIMPLEX, 0.6, (0,255,0), 2)

cap.release()
Test Cases
Tracking Concept Tests
P
persist=True in model.track() — why is it necessary?
Without persist=True, tracker state resets every frame call
persist=True keeps the Kalman filter state between calls, enabling ID continuity across frames
W
Object disappears behind another object for 10 frames
Tracker has no detection to match for 10 consecutive frames
DeepSORT keeps the track alive for max_age frames (default 30). Re-ID features help reassign the same ID when it reappears.
F
Two people cross paths — ID switch
Person A gets ID 1, Person B gets ID 2. After crossing, they swap.
ID switches are a known failure mode. DeepSORT's appearance embedding reduces this. For evaluation use MOTA and IDF1 metrics.

ML Fundamentals — Training, Testing and Evaluation
Datasets, splits, accuracy, overfitting vs underfitting
Machine LearningDatasetsOverfittingTrain/Test Split
▾

Machine learning is the process of finding patterns in data automatically, rather than hand-coding rules. A model learns from a training dataset and then generalises to new examples it has never seen. How well it does this is measured on a held-out test set.

Dataset Splits
Training set: 70–80%    Validation set: 10–15%    Test set: 10–15%
Training: model learns from these. Validation: tune hyperparameters here. Test: evaluate final model once only.
Overfitting
Model memorises the training data perfectly but performs poorly on new data. Training accuracy is high; validation accuracy is much lower. Fix: more data, data augmentation, dropout, early stopping.
Underfitting
Model has not learned enough from the training data. Both training and validation accuracy are low. Fix: more training epochs, larger model, reduced regularisation, better features.

Convolutional Neural Networks (CNNs)
How CNNs learn features from images, layers, filters and the classification workflow
CNNTensorFlowPyTorchConvolutionPooling
▾

A CNN automatically learns what features to look for in an image — edges, textures, shapes, object parts — through training. This is in contrast to hand-crafted features. Early layers detect low-level features (edges). Deeper layers detect complex patterns (faces, wheels, logos).

CNN Architecture
Input Image
(H, W, 3)
▶
Conv + ReLU
extract features
▶
Pooling
reduce size
▶
Flatten
1D vector
▶
Dense + Softmax
class scores
Convolutional Layer
A small filter (kernel, e.g. 3×3) slides across the image, computing a dot product at every position. This produces a feature map. Many filters run in parallel, each detecting a different pattern.
Pooling Layer
Reduces the spatial size (max pooling takes the maximum value in each region). This reduces computation, makes the network translation-invariant, and prevents overfitting by reducing parameters.
Dense + Softmax
After the convolutional layers, the feature map is flattened to a 1D vector and fed through fully-connected layers. The final softmax converts raw scores to a probability distribution over classes.
Output size after Conv: floor((W − K + 2P) / S) + 1
W = input size, K = kernel size, P = padding, S = stride  ·  Most standard: K=3, P=1, S=1 → output size = input size

Code — CNN Image Classification (TensorFlow/Keras)
Python — TensorFlow / Keras
import tensorflow as tf
from tensorflow.keras import layers, models

# Build a simple CNN for image classification
model = models.Sequential([
    layers.Conv2D(32, (3,3), activation='relu', input_shape=(224,224,3)),
    layers.MaxPooling2D((2,2)),
    layers.Conv2D(64, (3,3), activation='relu'),
    layers.MaxPooling2D((2,2)),
    layers.Flatten(),
    layers.Dense(128, activation='relu'),
    layers.Dropout(0.4),             # reduces overfitting
    layers.Dense(10, activation='softmax')  # 10 classes
])

model.compile(optimizer='adam',
              loss='sparse_categorical_crossentropy',
              metrics=['accuracy'])

model.summary()

# Train

CNN Architecture Tests
P
Conv output size: input=224, K=3, P=1, S=1
floor((224 - 3 + 2*1) / 1) + 1
Result: 224 — same size. Padding=1 with kernel=3 preserves spatial dimensions.
P
After MaxPooling(2,2) on 224×224 feature map
Each dimension halved
Result: 112×112. Pooling reduces spatial size, increases effective receptive field.
P
Trainable params in Conv2D(32, 3x3) on 3-channel input
params = (K*K*C_in + 1) * C_out = (3*3*3 + 1) * 32
Result: 896 parameters. The +1 accounts for bias per filter.
